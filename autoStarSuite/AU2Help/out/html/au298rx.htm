How To Manage Custom User Catalogs (Autostar II Only)

To Manipulate Custom User Objects, Revision 1.3o or higher of the Autostar II Software is required.

To transfer Custom User Objects from the Library List View to the Handbox List View or vice-versa, use the Transfer Buttons on the main dialog box in much the same way as they would be used with normal User Objects. However, due to potential variations between the source and destination catalog definitions, the following rules apply:

텲ven though catalogs may be given original names, they still maintain their numerical identities. For example, a catalog that had a default name of "Undefined13" before being defined will be treated as the 13th Custom Catalog. Thus, when either User Object type button is clicked, the 13th Custom Catalog will be displayed in both list views even if it is not defined identically in both list views.

텶f the source and destination list views have identical definitions of the catalog displayed, the "Transfer Selected" and "Transfer Refresh" buttons will work normally.

텶f the source list view catalog has been defined but the corresponding destination catalog is empty, the "Transfer Selected" buttons will create the catalog definition in the destination list before copying the objects, whereas the "Transfer Refresh" buttons will have no effect.

텶f the source list view catalog has a different definition than the destination catalog, neither the "Transfer Selected" and "Transfer Refresh" buttons will function.

톀egardless of the definitions of the destination list view catalogs, the "Transfer All" buttons will first erase all destination objects, then copy the source objects. Note that selecting "User Objects" in the "Confirm Object Replace" dialog will transfer ALL 20 Custom User Catalogs plus the normal User Objects.

톂he above rules apply to Real Time Mode as well.

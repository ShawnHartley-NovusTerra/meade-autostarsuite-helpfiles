How To Create Custom User Catalogs (Autostar II Only)

To Manipulate Custom User Objects, Revision 1.3o or higher of the Autostar II Software is required.

Before creating a Custom User Catalog, consider carefully the data that will be stored in that object. Each catalog can have 6 user-defined fields of 15 characters total, including the field name and data. For example, a field labeled "FIELDNAME" may have the data "123456" contained within it. The data is stored and displayed as pure character data, so decimal points are counted as a character, and any combination of letters and numbers and other characters may be used. For example, the above field may contain "123.4%". Of course, there is no minimum requirement except that the label is at least one character, so FieldName: "1" with data "" is acceptable. There are 3 characters that are not allowed in either field names or data. These are ‘:’, ‘#’, and ‘,’.

Custom User Catalogs may only be created from within the Autostar Update software. To do so, select an empty Custom User Object by right-clicking on the User Object type button in either list view. Empty catalogs are given a default name of "UndefinedX", where "X" is a sequential number between 1 and 20. Within the list view window, right-click to display the list view context menu. Select "new" from this menu to display the "Define New User Catalog" Dialog Box.

Within the dialog box, enter a name for this Catalog, and define up to 6 new fields by typing each field’s name in "Field Name:". As the name is typed, the number of characters available for data is displayed. The longer the field name, the shorter the data must be. Click "Add" to add each new field. When the catalog has been defined, click "OK" to proceed.

The "Autostar Update Object Editor" Dialog Box will now be displayed so object data may be entered. Note that there must be at least one object in a catalog, or its definition will be lost. To delete a custom catalog, simply delete all objects within that catalog.

For information on managing Custom User Objects including transferring between list views and the handbox, see Managing Custom User Objects.
